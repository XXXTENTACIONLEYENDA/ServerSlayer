## How to Contribute

### Reporting Bugs
- Open an issue with a clear title and description.
- Include steps to reproduce the bug.
- Mention your OS (Windows/macOS/Linux).

### Suggesting Features
- Open an issue with the `[Feature Request]` prefix.
- Describe the use case and expected behavior.

### Pull Requests
1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes.
4. Test locally by running `/listports` and `/killservers`.
5. Commit with clear messages.
6. Open a PR against `main`.

## Code Style
- Python: Follow PEP 8.
- Markdown: Use consistent headers and formatting.

## Areas to Contribute
- **macOS/Linux testing**: Help verify cross-platform compatibility.
- **New frameworks**: Add detection for more dev servers.
- **Idle detection**: Improve CPU/connection-based idle detection.
- **Documentation**: Better examples, GIFs, tutorials.

## Questions?
Open an issue or reach out!
